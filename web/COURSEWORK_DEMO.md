# Coursework Demo

## Strongest USP

MSIN helps students practise real business decisions, turns those choices into visible leadership skill signal, and then helps them connect with peers whose strengths complement their own.

Short version:

Practice decisions. Build leadership signal. Find the right peers.

## Three Demo Pathways

### 1. First-Time User

Use case:
A student wants to practise decision-making before a pitch, internship interview, or startup meeting.

Flow:
Landing -> onboarding -> first simulation -> feedback

What it proves:
The app creates value in one short session and does more than show static educational content.

### 2. Returning User

Use case:
A student comes back after one run and wants to improve a weaker skill such as communication or leadership.

Flow:
Dashboard -> identify weak skill -> retake simulation -> see score delta

What it proves:
The app supports progress over time, not just one-off assessment.

### 3. Network Discovery

Use case:
A student sees their strengths and then looks for a peer with complementary judgment for a project, society role, or startup idea.

Flow:
Dashboard -> network -> connect with a peer

What it proves:
Feedback leads to action, not dead-end analytics.

## Imagery Direction

Current implementation:
- Landing imagery uses Pexels-hosted photography
- Peer cards use portrait photography from Pexels

Recommended libraries for future iterations:
- Pexels for clean student and collaboration photography
- Unsplash for more editorial hero imagery
- unDraw for simple learning and onboarding illustrations

Guidelines:
- Use one photography style across the whole product
- Keep onboarding and landing imagery calm and editorial
- Use portraits only where identity or peer relevance matters
- Avoid mixing illustrations and photos in the same section
